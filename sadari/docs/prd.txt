# 팀 랜덤 선택기 (Team Random Selector) - 요구 사항 명세서 (PRD)

## 1. 프로젝트 목적
- 팀 내에서 회식 메뉴, 발표 순서, 당번 등을 공정하고 재미있게 정하기 위한 단순하고 직관적인 랜덤 선택기 웹 애플리케이션 개발.
- 모던하고 심플한 UI와 애니메이션으로 사용자 경험을 높이며, URL 및 텍스트 파일을 통해 쉽게 목록을 공유할 수 있도록 지원.

## 2. 기술 스택
- **Front-end**: Next.js (결과 페이지 및 라우팅 처리)
- **Styling**: Tailwind CSS (빠르고 일관된 모던 디자인)
- **Deployment & Hosting**: Cloudflare Pages
- **Language**: TypeScript 권장

## 3. 핵심 기능 (Core Features)

### 3.1. 참가자/항목 명단 입력 및 공유
- **URL 파라미터 공유**: `/?names=A,B,C` 와 같이 URL 파라미터로 데이터 입력 가능. 세팅된 URL을 팀원에게 전달하여 동일한 명단으로 즉시 시작 가능.
- **텍스트 파일 업로드**: 참가자가 많은 경우 `.txt` 파일을 업로드하여 간편하게 명단 설정.
- **수동 텍스트 입력**: 화면 내 Textarea를 통해 직접 입력 (쉼표 또는 줄바꿈으로 구분).

### 3.2. 랜덤 추첨 기능
- **단일 항목 뽑기**: 입력된 전체 항목 중 1개 (혹은 지정된 N개)를 랜덤으로 선택 (예: 회식 메뉴 고르기, 오늘의 당번).
- **순서 섞기**: 입력된 리스트의 순서를 랜덤하게 섞어서 배치 (예: 발표 순서 정하기).

### 3.3. UI/UX 및 애니메이션
- **모던 & 미니멀리스틱 디자인**: 군더더기 없는 깔끔한 레이아웃 (Dark mode 지원 고려).
- **역동적인 애니메이션 효과**: 
  - 추첨 시 텍스트 롤링/슬롯머신/룰렛 등의 효과로 긴장감 부여
  - 결과 발표 시 폭죽(Confetti) 효과 등 시각적 즐거움 제공
- **반응형 레이아웃**: 스마트폰 및 PC 환경에 모두 최적화 (링크로 공유받고 모바일로 접속할 확률이 높음).

## 4. 사용자 플로우 (User Flow)
1. **접속 및 초기화**: 
   - 도메인 접속 시 URL 파라미터(`?names=...`)가 있으면 자동으로 화면에 항목 초기화. 
   - 파라미터가 없으면 직접 입력 혹은 파일 업로드 창 제공.
2. **모드 선택**: '한 명 뽑기' 또는 '순서 섞기' 모드 선택.
3. **추첨 시작**: 버튼을 누르면 애니메이션과 함께 추첨 시작 (약 2~3초 지속).
4. **결과 확인**: 명확한 강조(Highlighting) 효과와 함께 결과 노출.
5. **재추첨/수정**: 기존 목록에서 이미 뽑힌 사람을 제외하고 재추첨하거나, 처음으로 돌아가 목록 수정 기능 제공.

## 5. 인프라 설계
- Next.js 정적 빌드(Static Export) 또는 Edge Runtime을 활용하여 Cloudflare Pages로 배포.
- 별도의 DB/백엔드 서버 없이 프론트엔드 레벨과 URL 상태(State)만으로 동작하는 무상태(Stateless) 아키텍처.

## 6. 개발 마일스톤 (진행 순서 제안)
- [Step 1] Next.js 뼈대 구축 및 Tailwind CSS, Cloudflare 배포 환경 세팅
- [Step 2] 명단 파싱 및 상태 관리 로직 (`names` URL 파라미터 연동, 파일 업로드 기능) 구현
- [Step 3] 핵심 랜덤/셔플 로직 작성
- [Step 4] UI 컴포넌트(입력 폼, 결과 표시 등)와 애니메이션 효과 이식 및 연결
- [Step 5] 전체 테스트 및 QA 진행, 최종 수정 후 배포 완료
